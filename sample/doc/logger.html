Module logger

creates a logger class.

this class does not throw exceptions directly, note return values and error messages need to be dealt with in your own code. designed for use with the classy object creator

Usage:

    Logger = require ( 'logger ' )

initMethod (obj)
adds the default log levels Error, Warning, Info and Debug and sets their associated default values ON (true), ON (true), OFF (false), OFF (false). also sets the logger to the default output.

Parameters:

  • obj the calling object

Returns:

    No return value

See also:

    setMyOutput

Usage:

  • myLogger = Logger ()
  • myLogger = Logger ( { [ callLogOutput = YOUR_OWN_LOG_FUNCTION,
                       masterLogState = true | false,
                       myOwnLogState = true | false ]
                   })
addLogLevel (obj, logLevel)
addLogLevel, adds a log level completly from the logger. All registered modules will also recive this log level

Parameters:

  • obj the calling object OR class itself
  • logLevel the level you want to add

Returns:

    true if success, false if failed - Failed also returns an error message

Usage:

    logger:addLogLevel ( 'LOG_LEVEL_NAME' [, true | false ] )

log (obj, logLevel, ...)
log, test the master state and if OFF (false) stop all further log processing the default use of log calls at the Global level, i.e. no module name provided

Parameters:

  • obj the calling object OR class itself
  • logLevel the log level
  • ... the items to log

Returns:

    true if success, false if failed - Failed also returns an error message, note that if called with no values it always returns the name of the logLevel called

Usage:

  • logger:log ( 'LOG_LEVEL_NAME' | logLevel, [ ... Items you want to log in a list ] )
  • Log_LOG_LEVEL_NAME ( [ ... Items you want to log in a list ] ) -- note this calls up to _G scope
logFromModule (the, moduleName, logLevel, ...)
logFromModule, test the master state and if OFF (false) stop all further log processing the default use of log calls at the Global level, i.e. no module name provided

Parameters:

  • the calling object OR class itself
  • moduleName the module name you want the log to come from
  • logLevel the log level
  • ... the items to log

  • module.Log_LOG_LEVEL_NAME ( [ ... Items you want to log in a list ] ) -- note this calls up to the module scope
setLogState (the, logState, ...)
setLogState, sets a specific log value to true (on) or false (off) if called by an object then that state ONLY applies to that object, if called by the class itself then it applies to ALL objects, except those that have a specific state note if master is off ( false ) then even objects that have their own log level of true WILL NOT LOG. to log the global master MUST be ON, the local master log level must be ON and your object specific value must be ON

Parameters:

  • the calling object OR class itself
  • logState the specific state you want true or false (must be a bool)
  • ... the various levels you wish to change

Returns:

    true if success, false if failed - Failed also returns an error message

Usage:

    logger:setLogState ( true | false [, LIST OF LOG LEVELS TO APPLY TO ] )
setModuleLogState (obj, moduleName, logState, ...)
setModuleLogState, sets a specific log value for a module to true (on) or false (off) if called by an object then that state ONLY applies to that object, note if master is off ( false ) then even objects that have their own log level of true WILL NOT LOG. to log the global master MUST be ON, the local master log level must be ON and your object specific value must be ON

Parameters:

  • obj the calling object OR class itself
  • moduleName the calling object OR class itself
  • logState the specific state you want true or false (must be a bool)
  • ... the various levels you wish to change

Returns:
